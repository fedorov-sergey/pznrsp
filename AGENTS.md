## Стек и технологии

8. Фреймворк — **Bootstrap 5** (CDN), иконки — **Bootstrap Icons** (CDN). Не подключать сторонние CSS/JS-библиотеки без необходимости.
9. Все стили — в `<style>` внутри `<head>` конкретной страницы. Глобальный CSS-файл не используется.
10. Подход к верстке — **блочный** (как в Тильде): каждая секция страницы — это отдельный `<section>` с собственным фоном/отступом. Это обеспечивает визуальную целостность и адаптивность.

## Структура каждой страницы (каркас)

Каждая новая страница должна повторять этот каркас:

```
1. <nav>         — тёмный navbar (bg-dark), логотип «Анализ смет», меню-бургер на мобильных, offcanvas-menu
2. <section>     — hero/header — градиентный заголовок (linear-gradient #0d6efd -> #0a58ca)
3. <section>     — основной контент (py-5, белый или серый фон, чередование)
4. <footer>      — тёмный футер (bg-dark, серый текст), ссылка «На главную»
```

- Навигация и футер — **идентичны** на всех страницах.
- При добавлении новой страницы — добавлять пункт в navbar (и обычное меню, и offcanvas-меню).
- Hero-секция обязательна для заглавных страниц. Для внутренних — допустим заголовок без градиента.

11. Использовать Bootstrap-классы: `col-lg-*`, `col-md-*`, `d-flex`, `d-grid`, `d-md-flex`, `w-100`, `gap-*`.

## Страницы парсеров (генерация)

Страницы парсеров генерируются автоматически из Python-файлов приложения.

### Структура

```
build/
├── generate.py               ← скрипт-генератор
└── template_parser.html      ← шаблон страницы парсера
parsers/
├── LP17_Koeff_parsers.html   ← ~95 сгенерированных страниц
└── pic/                      ← скриншоты
```

### Как менять дизайн

Правишь `build/template_parser.html` → запускаешь `python build/generate.py` → все страницы обновлены.

### Как запускать

```
cd build
python generate.py
```

### Что делает скрипт

1. Сканирует `.py` файлы парсеров из исходного кода приложения
2. Извлекает метаданные через `ast`: NAME, DESCRIPTION, PREMIUM, PIC, PATH, COLUMNS
3. Сохраняет промежуточный `parsers.json`
4. Генерирует HTML из шаблона, подставляя данные
5. Копирует скриншоты в `parsers/pic/`
6. Генерирует каталог `parsers.html` через функцию `generate_catalog_page` (аккордеон-группировка по PATH, формат «пилюль», поиск, кнопки свернуть/развернуть)
7. Обновляет `sitemap.xml`
